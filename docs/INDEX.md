## üìù Documentation Standards

All Nexus HR documentation follows these standards:

- **Format**: Markdown (GitHub Flavored)
- **Structure**: Clear headings, table of contents for long docs
- **Code Examples**: Syntax highlighted with language tags
- **Cross-References**: Relative links to other docs
- **Versioning**: Dated at bottom when significant updates occur
- **Clarity**: Written for developers of all experience levels

---

## üîÑ Keeping Documentation Updated

Documentation is a living resource. When making changes:

1. **Code Changes**: Update relevant docs in the same PR
2. **API Changes**: Update API.md and integration docs
3. **Architecture Changes**: Update ARCHITECTURE.md
4. **New Features**: Update README.md and relevant guides
5. **Bug Fixes**: Update TROUBLESHOOTING.md if applicable
